HardView
Your Window into Hardware Information

A high-performance, cross-platform Python library providing detailed hardware and system information in JSON format.

Install Now Usage Examples

Key Features

Comprehensive Hardware Data

Get detailed information about BIOS, System, Motherboard, CPU, RAM, Disks, and Network.

Advanced Storage & SMART

Detailed disk, partition, and advanced SMART attributes.

Performance Monitoring

Monitor CPU, RAM, and system performance in real-time and over intervals.

Cross-Platform

Works seamlessly on Windows (via WMI) and Linux (via sysfs/proc) systems.

C Implementation (Ultra-Fast)

Built in C to ensure the highest performance and minimal resource consumption.

Structured JSON Output

All results are returned as JSON strings for easy parsing and integration.

HardView is significantly faster than other Python libraries due to its C implementation and direct system API access.

Installation

Install from PyPI (Recommended)

The easiest way to install HardView is via pip from the Python Package Index (PyPI): 

pip install hardview

HardView supports Windows (32/64-bit) and Linux (x86/x86_64) systems.

Install from Source

If you want to build HardView yourself (for development or unsupported Python versions): 

git clone https://github.com/gafoo173/hardview.git
cd hardview
pip install .

For source installation on Windows, ensure you have a working C compiler (Visual Studio Build Tools recommended).

Usage Examples

HardView is incredibly easy to use. Here's a simple example showing how to get CPU information: 

import hardview
import json
import pprint # For pretty printing JSON output

# Get CPU information
cpu_info_json = hardview.get_cpu_info()

# Parse the JSON string into a Python object
cpu_info = json.loads(cpu_info_json)

# Pretty print the information
print("CPU Information:")
pprint.pprint(cpu_info)

# Another example: Get RAM information
ram_info_json = hardview.get_ram_info()
ram_info = json.loads(ram_info_json)
print("\nRAM Information:")
pprint.pprint(ram_info)

# Example of monitoring CPU usage for 5 seconds with a 1-second interval
print("\nStarting CPU usage monitoring for 5 seconds...")
cpu_monitor_data = json.loads(hardview.monitor_cpu_usage(5, 1000))
print("CPU Monitoring Data:")
pprint.pprint(cpu_monitor_data)

Always remember to use json.loads() to parse the output from HardView, as it returns JSON strings.

When to Use HardView?

HardView is the ideal choice when you need:

Detailed, accurate, and complete hardware information (including BIOS, baseboard, chassis, SMART, etc.).

Extremely fast queries (with native C execution speed).

Real-time or interval-based performance monitoring.

Consistent and structured JSON output for easy integration.

A single cross-platform solution for both Windows and Linux.

Frequently Asked Questions

Q1: pip install fails or cannot find a suitable wheel?

A: Ensure you are using a supported platform (Windows 32/64-bit or Linux x86/x86_64) and a supported Python version. Upgrade pip to the latest version: pip install --upgrade pip. If no wheel is available, try building from source.

Q2: HardView does not work after installation or shows an ImportError?

A: Ensure the library file in your site-packages directory is named HardView.pyd (or HardView.so on Linux) with correct casing. Make sure to import the library using import HardView. Reinstall if necessary.

Q3: Does HardView support Linux, macOS, or other platforms?

A: HardView supports Windows (32/64-bit) and Linux (x86/x86_64) only. There is no support for Arch Linux, Android, or macOS.

Q4: Why is HardView faster than other Python libraries?

A: HardView is written entirely in C and interacts directly with system APIs, minimizing overhead and maximizing speed.

Q5: Is HardView thread-safe?

A: HardView is not inherently thread-safe. If you use it in multi-threaded code, ensure calls are serialized or protected by the Python GIL.
